# 需求說明書（Software Requirements Specification, SRS）

## 1. 專案簡介
本專案為「小說與漫畫閱讀網站」之系統設計規劃，主要目的在於提供使用者一個可線上瀏覽、搜尋與閱讀小說及漫畫的平台。本文件著重於系統需求與設計思路之說明，並未要求完成實際系統成品。

---

## 2. 使用者角色（Actors）

| 角色 | 說明 |
|---|---|
| 訪客 | 未登入之一般使用者 |
| 會員 | 已註冊並登入之使用者 |
| 作者 | 可新增與管理作品之使用者 |
| 管理員 | 負責系統管理與內容審核 |

### UML 對應
- 用例圖（Use Case Diagram）

---

## 3. 功能需求（Functional Requirements）

### 訪客
- 瀏覽作品
- 搜尋作品
- 閱讀小說與漫畫內容

### 會員
- 註冊與登入
- 收藏作品
- 紀錄閱讀進度

### 作者
- 新增作品
- 編輯小說章節
- 上傳漫畫圖片

### 管理員
- 使用者管理
- 作品審核
- 刪除違規內容

📌 本節需求對應 UML 用例圖。

---

## 4. 非功能需求（Non-functional Requirements）

| 項目 | 說明 |
|---|---|
| 效能 | 系統操作需於合理時間內回應 |
| 安全性 | 使用者密碼需加密處理 |
| 可用性 | 介面設計需簡單易懂 |
| 擴充性 | 系統可支援未來功能擴充 |

---

## 5. 介面需求
- 首頁：作品列表、搜尋功能
- 閱讀頁：顯示小說或漫畫內容
- 管理頁：後台管理介面
