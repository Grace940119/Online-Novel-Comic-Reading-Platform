# 概要設計說明書（High-Level Design, HLD）

## 1. 系統架構設計
本系統採用前後端分離架構，將使用者介面、系統邏輯與資料儲存分開設計，以提升系統維護性與擴充性。

### 架構組成
- 前端：負責使用者操作與畫面顯示
- 後端：負責業務邏輯與資料處理
- 資料庫：儲存系統相關資料

### UML 對應
- 系統架構圖（System Architecture Diagram）

---

## 2. 系統模組劃分

| 模組 | 功能說明 |
|---|---|
| 使用者模組 | 註冊、登入與權限管理 |
| 作品模組 | 小說與漫畫作品管理 |
| 閱讀模組 | 內容顯示與進度記錄 |
| 管理模組 | 審核與刪除作品 |

### UML 對應
- 套件圖（Package Diagram）

---

## 3. 資料設計概念
系統主要資料包含使用者、作品、章節與閱讀紀錄等，資料間之關聯以關聯式資料模型設計。

### UML 對應
- ER 圖（Entity Relationship Diagram）
