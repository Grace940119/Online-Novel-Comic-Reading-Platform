# 詳細設計說明書（Detailed Design Document, DDD）

## 1. 類別設計概念
系統核心類別包含：
- User：使用者資料與角色資訊
- Work：作品基本資料
- Chapter：小說章節內容
- ComicImage：漫畫圖片資料
- ReadingRecord：閱讀進度紀錄

### UML 對應
- 類別圖（Class Diagram）

---

## 2. 系統互動流程設計

### 範例：會員閱讀小說流程
1. 會員登入系統
2. 選擇作品
3. 選擇章節
4. 系統顯示內容
5. 系統自動紀錄閱讀進度

### UML 對應
- 循序圖（Sequence Diagram）
- 活動圖（Activity Diagram）

---

## 3. 邏輯與演算法設計（概念性）
- 閱讀進度以使用者與章節關聯方式記錄
- 收藏功能以使用者與作品建立關聯
- 權限控管依角色限制功能操作

📌 本文件僅說明設計邏輯與思路，未涉及實際程式碼實作。
