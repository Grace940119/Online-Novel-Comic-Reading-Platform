1. 專案簡介
本專案為「小說與漫畫閱讀網站」之系統設計規劃，目的在於提供使用者一個可線上瀏覽、搜尋與閱讀小說與漫畫內容的平台。本文件僅說明系統之需求與設計思路，並未要求完成實際成品。

2. 使用者角色（Actor）
   
| 角色  | 說明        |
| --- | --------- |
| 訪客  | 未登入使用者    |
| 會員  | 已註冊登入之使用者 |
| 作者  | 可上傳作品之使用者 |
| 管理員 | 負責系統管理與審核 |

四、測試計畫（Test Plan）
📌 回答「如何確認系統是正確的」

1. 測試目標
確保系統各功能符合需求說明書所定義之內容。

2. 測試範圍
測試項目	說明
功能測試	驗證用例是否可正常執行
使用者測試	確認操作流程合理
權限測試	不同角色功能限制
3. 測試方法
依據 用例圖 設計測試案例

每個用例至少一個測試情境

4. 測試資源配置
測試人員：開發者本人

測試工具：瀏覽器、模擬資料
